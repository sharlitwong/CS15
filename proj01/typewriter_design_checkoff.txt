4. Consider implementing the handling of the input "backspace." What
are the different cases your implementation must consider? What
should your implementation do in each case; specifically, how are the
inputs to TextUI's render() function altered? You should play around
with the reference implementation when coming up with your answers.

Answer:

Different cases I'm considering for the input "backspace" include:
1. The cursor is located at column 0, line 0; there is nothing to be deleted.
   Cursor's location after delete is pressed: same as where it was originally,
   column 0 & row 0.
2. The cursor is located at column 0, but not row 0. In this case, the line
   immediately after the cursor would no longer be a "new line" after the user
   pressed delete, and would follow after the last character in the preceding 
   line. 
   Cursor's location after delete is pressed: the row should decrease by 1, and
   the column should be the length of the original previous line (before merge)
   modded by the terminal width.
3. Cursor is anywhere other than the other two cases. In this case, the 
   character immediately preceding the cursor's current location should be 
   deleted. 
   Cursor's location after delete is pressed: the cursor row should remain the 
   same while the column should decrease by 1.

I need to instantiate two ActionStack objects: an undo stack, and a redo stack.

Undo Stack:
Actions will be pushed to this stack when the user does anything to alter their
file (i.e. typing anything or pressing redo).

Redo Stack:
Actions that are undone will be pushed to this stack, as they're the only 
actions that could possibly be redone. This stack will be cleared anytime the 
alters the text excluding via the undo function, as redoing is not possible if 
nothing was undone.

8. Consider implementing the "up" arrow key functionality.

   (a) What functions from the TextUI class will you need to use to
       implement "up"?

Answer:
      I would need render() and and getTerminalWidth(). Render would give me the
      current state of the file, including where the cursor is, and all the  
      existing lines. Then, getTerminalWidth would allow me to tell whether any
      given line wraps or not.

   (b) What are the different cases you need to consider for "up"?
       In particular, consider: different lengths of the current line
       (including lines that are long enough to wrap around), where
       the cursor is located, and different lengths for the line prior
       to the current line.

Answer:

       Case 1: the cursor is on the first logical line, which has yet to wrap 
       around. In this case, nothing should be done, as the cursor cannot move
       further up. 
       Case 2: the cursor is in some logical line, which is long enough to wrap 
       to the next line. The cursor should move up one visual line, but 
       remain within the same logical line. 
       Case 3: the cursor is in a line that doesn't wrap, and there exists some
       line above which doesn't extend as far horizontally as the cursor's 
       current location. The cursor should move up one logical line, to the end
       of the previous logical line.
       Case 4: the cursor is in some line, that has yet to wrap, an there exists
       some line above it that extends horizontally past where the cursor is 
       currently located. In this case, the cursor should move up one logical 
       line and remain in its original column.

   (c) Provide pseudocode for the handling of the "up" arrow.

   t_Width = getTerminalWidth()
   colInVisualLine = cursorCol % termWidth
   visualLineOfCursor = cursorCol / termWidth

   if cursorLine == 0 and visualLineOfCursor == 0:
      do nothing  //case 1

   else if visualLineOfCursor > 0:
      //case 2: wrap within same logical line
      cursorCol -= termWidth

   else:
      //cases 3 and 4: move to previous logical line
      cursorLine -= 1
      prevLineLen = lines[cursorLine].length()
      lastVisualLineStart = (prevLineLen / termWidth) * termWidth
      newCol = lastVisualLineStart + colInVisualLine
      if newCol > prevLineLen:
         newCol = prevLineLen  //case 3: line too short
      cursorCol = newCol        //case 4: line long enough
